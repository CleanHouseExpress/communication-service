# Orchestra Events Contract

This document describes the internal event contract expected from `orchestra-api`.

All events are sent to:

`POST /api/internal/orchestra/events/tenants`

## Headers

```http
Accept: application/json
Content-Type: application/json
X-Service-Token: <service-token>
```

`Authorization: Bearer <service-token>` is also accepted.

Secrets must not be placed in event payloads.

## Common Envelope

```json
{
  "event_id": "evt_123",
  "event_type": "TenantCreated",
  "occurred_at": "2026-06-24T15:00:00-03:00",
  "tenant": {
    "id": "tenant_123",
    "name": "Rede Exemplo",
    "slug": "rede-exemplo",
    "status": "active",
    "timezone": "America/Sao_Paulo",
    "metadata": {}
  }
}
```

Fields:

- `event_id`: stable id generated by Orchestra. Required and idempotent.
- `event_type`: event name. Required.
- `occurred_at`: event time. Optional but recommended.
- `tenant`: tenant payload. Required for known tenant events.

## Idempotency

The communication-service stores each event in `communication_integration_events`.

Idempotency key:

- `source=orchestra-api`
- `event_id`

If the same event is delivered again, the service returns the existing event and does not apply tenant changes a second time.

## TenantCreated

Creates or updates a local tenant replica as active unless a valid `tenant.status` is provided.

```json
{
  "event_id": "evt_tenant_created_123",
  "event_type": "TenantCreated",
  "occurred_at": "2026-06-24T15:00:00-03:00",
  "tenant": {
    "id": "tenant_123",
    "name": "Rede Exemplo",
    "slug": "rede-exemplo",
    "status": "active",
    "timezone": "America/Sao_Paulo",
    "metadata": {
      "source": "orchestra"
    }
  }
}
```

## TenantUpdated

Updates the local tenant replica. If `tenant.status` is omitted, the status defaults to active.

```json
{
  "event_id": "evt_tenant_updated_123",
  "event_type": "TenantUpdated",
  "occurred_at": "2026-06-24T15:10:00-03:00",
  "tenant": {
    "id": "tenant_123",
    "name": "Rede Exemplo Atualizada",
    "slug": "rede-exemplo",
    "timezone": "America/Sao_Paulo",
    "metadata": {}
  }
}
```

## TenantDisabled

Marks the local tenant replica as disabled. The payload status is ignored and mapped to `disabled`.

```json
{
  "event_id": "evt_tenant_disabled_123",
  "event_type": "TenantDisabled",
  "occurred_at": "2026-06-24T15:20:00-03:00",
  "tenant": {
    "id": "tenant_123",
    "name": "Rede Exemplo",
    "slug": "rede-exemplo",
    "timezone": "America/Sao_Paulo",
    "metadata": {
      "reason": "administrative_disable"
    }
  }
}
```

## TenantEnabled

Reactivates the local tenant replica and maps status to `active`.

```json
{
  "event_id": "evt_tenant_enabled_123",
  "event_type": "TenantEnabled",
  "occurred_at": "2026-06-24T15:30:00-03:00",
  "tenant": {
    "id": "tenant_123",
    "name": "Rede Exemplo",
    "slug": "rede-exemplo",
    "timezone": "America/Sao_Paulo",
    "metadata": {}
  }
}
```

## Unknown Events

Unknown event types are stored with status `ignored`. They do not update tenant replicas.

## Response

```json
{
  "integration_event_id": "uuid",
  "event_id": "evt_123",
  "status": "processed",
  "tenant_id": "uuid",
  "idempotent": false
}
```

Statuses:

- `processed`: known event applied.
- `ignored`: unknown event stored but ignored.
- `failed`: event stored, but processing failed.

## Current Limitations

- No physical tenant database is created.
- No queues or retries.
- No dynamic connection switching.
- No user, RBAC, TBAC or permissions replication.
- No events are published back to Orchestra.
